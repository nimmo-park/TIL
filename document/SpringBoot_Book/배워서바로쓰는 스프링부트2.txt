스프링 부트의 대표적특징
- 스타터 : 의존관계를 간단하게 정의하는 모듈이다. 
- 빌드 도구 : 버전 해결 등 개발을 효율화하는 플러그인이다.
- 구성 클래스 : XML이 아닌 애너테이션과 자바로 설정을 작성한다
- 메인 애플리케이션 클래스 : 자바명령으로 내장된 톰캣을 실행한다.
- 설정파일 : 속성을 외부파일에 정의할 수 있으며 동작 사양을 쉽게 변경 할 수 있다.

* spring-boot-stater-web : 스프링 MVC, 톰캣이 의존관계에 추가된다.
* spring-boot-stater-jdbc: 스프링 JDBC, 톰캣 JDBC 커넥션 풀이 의존관계에 추가가 된다.

스프링부트는 아파치 메이븐과 그레이들을의 사용을 권장한다.

-그레이들은 스크립트를 작성하는 빌드도구이므로 아파치 앤트처럼 작업을 자유롭게 작성가능
-멀티플로젝트 구성시에 하위 프롤젝트에 대해 일괄로 설정하고 필요에 따라서 개별적으로 설정가능 스크립트의 작성량이 메이븐보다 적다.
-메이븐은 특수한 처리가 필요할 때 독자적인 플러그인으로 구현해야 하지만 그레이들은 스크립트를 작성하는 것만으로도 대응가능하다.

- spring boot는 devtools 모듈을 제공한다. JVM핫 스와핑과는 다른 방식으로 애플리케이션 개발의 효율성을 올려준다.

디폴트 속성
- 템플릿 엔진등 라이브러리에서는 캐시를 사용하여서 성능향상을 도모한다. 프로덕션 환경에서는 캐시가 필수지만 개발계에서는 필수가 아니다. 개발게에서는 애플리케이션이 재실행되어야하기때문에 오히려 방해요소가 된다. 그래서 devtools를 통해서 프로퍼티에 따로 작성하면 알아서 자동으로 적용이된다.

재실행시 트리거에서제외할 파일설정하기
예시 ) spring.devtools.restart.exclude = static/**,public/**

재실행 트리거에서 제외할 파일 추가 설정하기
예시 ) spring.devtools.restart.additional-exclude = java/**,test/**

LiveReload
- css javascript 등 템플릿이 변경되어서 브라우저가 새로고침 자동으로되어서 적용된화면을 보여준다.
예시 ) spring.devtools.livereload.enabled = false
